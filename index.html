Autra.ai Website Redevelopment

The Problem

Autra.ai, an AI-driven education company serving children aged 5–12, required a fast and modern website overhaul. The previous website had outdated content, poor UI/UX, and lacked integration with our customer management platform, WellnessLiving. These issues hindered our ability to sign up and retain customers efficiently.

Project Goals

  • Rapidly deploy a new marketing and scheduling platform for business continuity
  • Improve user experience and update branding
  • Enable direct customer bookings via WellnessLiving integration
  • Ensure maintainability and scalability of the codebase using modern frontend tooling

Details

Implementation Highlights

  • Frontend Framework: Built with Vite.js and React, ensuring fast local dev environment and optimized builds.
  • UI Design: Used TailwindCSS to create a clean, responsive, and mobile-friendly layout.
  • Database Setup: Set up schema and typed queries using Drizzle ORM, improving developer productivity and safety.
  • Tooling: Developed within Replit using Replit Agent, speeding up code generation and testing workflows.
  • Bookings Integration: Embedded direct booking links and smart redirects to WellnessLiving, streamlining the scheduling process.
  • DevOps & Deployment: Deployed the final website to a production domain via Replit's deployment tooling and custom scripts.

Architecture Overview

  • drizzle.config.ts: Drizzle ORM setup for managing typed queries and schema
  • vite.config.ts: Custom configuration for optimized builds
  • tailwind.config.ts: Design token and theme customization
  • components/, pages/, routes/: Modular component and routing structure using file-based routing and layout wrappers
  • public/: Static assets for meta tags, branding, and icons

Challenges and Solutions

  • Rapid Launch Timeline: Balanced speed and quality by leveraging Replit Agent and strong component architecture
  • Integration with External System (WellnessLiving): Used deep linking and consistent UX wrappers to maintain brand presence even when directing off-site
  • Content Changes from Non-Tech Stakeholders: Created structured theme and layout JSON configs to easily adapt without redeploying

Impact and Conclusion

  • New website deployed in under 2 weeks
  • Significant improvement in navigation and mobile usability
  • Re-enabled business transactions and class signups directly through the site
  • Increased brand presence through a cleaner, more professional web presence

This project demonstrated my ability to rapidly build and deploy a modern, production-ready website under tight business constraints. By combining my technical skills in React, Vite, Drizzle ORM, and Tailwind with practical problem-solving and communication with non-technical stakeholders, I delivered a solution that met real business needs. It also marked my first time working with third-party tooling like WellnessLiving and using AI pair programming via Replit Agent, giving me experience in adapting to new tools and ecosystems quickly.

USEL Tournament Management Web Application

Overview

At USEL (United States Engineering League), I co-developed a web application aimed at streamlining the organization and management of robotics tournaments, specifically aligning with the requirements of the World Robot Olympiad (WRO). The application was designed to facilitate the creation, tracking, and administration of tournaments across multiple game modes, enhancing the efficiency of event management for organizers and participants.

The Problem

Organizing robotics tournaments at scale often involved fragmented tools (Google Sheets, manual score tracking, paper brackets). USEL needed a centralized platform to:

  • Allow users to create and manage tournaments across multiple game modes
  • Automate team and bracket management
  • Support tournament media uploads and updates
  • Be intuitive enough for non-technical users (e.g., coaches, teachers)

Details

Detailed insights into system architecture, design, and implementation processes.

Backend (Django)

  • Modeled core entities: Tournaments, Teams, Players, Game Modes, Scores, Brackets
  • Enabled CRUD operations for all entities using Django views and templates
  • Designed relational structure to support multiple game types:
    • Time-Based
    • Score-Based
    • Hybrid Mode (time + score)
  • Used Django Admin and ORM to manage internal content efficiently

Frontend (Bootstrap + HTML/CSS/JS)

  • Created a responsive UI using Bootstrap.
  • Implemented interactive UI components like modals, carousels, and dynamic bracket views
  • Ensured consistent styling across pages (homepage, tournament dashboards, team pages)

Routing and URL Mapping

  • URL Configuration: Defined clear and RESTful URL patterns in Django to map views to specific endpoints, facilitating intuitive navigation and API access.
  • View Functions: Developed corresponding view functions to handle requests and responses, processing data and rendering appropriate templates or JSON responses.

Version Control

  • Git Integration: Employed Git for version control, managing codebase changes, collaborating with team members, and maintaining a clear commit history

Impact and Conclusion

  • Operational Efficiency: Replaced spreadsheets and manual systems with a single web app used internally by USEL for tournament prep.
  • Scalability: Provided a foundation for future enhancements, including real-time scoring, analytics dashboards, and role-based permissions
  • User Adoption: Received positive feedback from tournament organizers and participants for its intuitive interface and reliability, leading to widespread adoption within USEL events.

This project helped me sharpen my full-stack web development skills in a real-world, team-based setting. I gained hands-on experience in:

  • Django MVC design and custom model relationships
  • Building reusable, responsive UI with Bootstrap
  • Designing software to support structured educational competitions
  • Collaboration and version control workflows using Git

    AI Art Generation System

    Overview

    Self-directed project explored AI-generated art using diffusion models and modular visual pipelines. I built a system for generating high-quality, stylized images by customizing workflows within ComfyUI, a node-based interface for Stable Diffusion. The goal was to understand how LoRA models, base model architecture, and visual prompt workflows could be optimized for flexible, reproducible image generation.

    The Problem

    Generative AI art tools like MidJourney or DALL·E provide limited customization or reproducibility. My goal was to build a highly customizable AI art pipeline that:

    • Gives fine-grained control over prompt input, style, and generation parameters
    • Enables workflow modularity for experimentation with base models and LoRA adapters
    • Supports repeatable generation using consistent seed and sampling setups
    • Allows easy expansion with new models, prompts, and visual effects

    Breakdown

    The project dives into model tuning, prompt engineering, and creative adjustments.

    Environment Setup & Tooling

    • Installed and configured ComfyUI, a node-based UI for building image generation workflows.
    • Integrated Stability Matrix for stable diffusion model access and workflow template management.
    • Loaded base models (e.g., Stable Diffusion v1.5, v2.1) into the runtime environment

    Workflow Creation

    • Designed custom generation pipelines by chaining visual nodes (e.g., Text Prompt → Sampler → Model Loader → Output).
    • Incorporated LoRA (Low-Rank Adaptation) workflows for injecting stylized weights into base models, allowing domain-specific generation without full retraining

    Prompt Engineering & Testing

    • Iteratively tested sampling methods (Euler, DPM++), CFG scale, denoising, and seed variation to evaluate output quality and reproducibility.
    • Tuned LoRA strength and conditioning values to blend styles without overfitting
    • Repeatable Outputs: Used fixed seeds and prompt templates to ensure image regeneration and comparison.

    Impact and Conclusion

    This side project gave me hands-on experience with real-world AI tooling and workflow engineering. It sharpened my skills in system design, modular development, and AI tool integration—all in a context where experimentation and optimization were key. It reflects my drive to not just use powerful AI tools, but understand and shape the systems behind them. I’m especially excited to apply this kind of modular thinking and system design in software development, creative tools, or applied AI projects.

    Learnings

    • Gained practical understanding of diffusion models, prompt engineering, and modular UI-based AI pipelines.
    • Deepened technical knowledge in model interoperability, visual UI node architecture, and GPU resource management.
